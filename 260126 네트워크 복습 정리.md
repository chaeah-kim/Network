# 네트워크 (Network)
* 한 장비에서 다른 장비를 통신매체로 연결해서 정보나 자료를 전달하는 조직이나 망을 의미
* 정보와 자료, 자원의 공유가 훨씬 편해졌다


**네트워킹이란?**
서로 연결하는 것 (그냥 연결만 한다고 네트워킹이 되는 것은 아니다)
-> 서로 연결된 장비들끼리 대화를 주고받을 수 있어야 한다 -> 정보의 공유를 위하여  
<br>


- 인터넷에 대하여
  
 Internet의 <Inter> -> 연결을 의미함
 ex) International → 국제적인, 국제간의 → 여러 나라를 묶을 때 사용하는 말

Interpol, Interphone → 각 나라의 경찰을 묶은 것, 전화를 서로 묶어주는 역할 등등

⇒ Internet이란 여러 개의 네트워크를 묶었다 는 의미를 가지고 있음.

전세계의 수많은 LAN과 WAN들이 연결된 거대한 네트워크

각각의 회사나 단체에서 자신들의 정보를 공유하고자 만들었던 네트워크를 좀 더 많은 사람들과 정보를 공유하고자 서로 연결하기 시작한 것이 바로 인터넷의 시작이 됨.

(Web → 맨 앞에 붙는 www → World Wide Web의 약자. 거미줄처럼 인터넷이 서로 연결되어 있기 때문에 거미줄을 뜻하는 말을 씀)

네트워크가 확장됨에 따라 연결된 네트워크간에는 서로 동일한 **프로토콜**을 사용해야 할 필요가 생김

→ 프로토콜(protocol)은 언어,규칙으로 비유할 수 있다  

<br>

## 프로토콜 
서로 다른 네트워크가 통신을 하기 위한 언어 혹은 약속
서로 연결된 네트워크는 같은 규칙을 사용해야 한다
(대표적인 네트워크 프로토콜로 인터넷 환경에서 데이터를 전송하는  TCP/IP 가 있음)

**구성범위에 따른 네트워크 분류**
* LAN (Local Area Network)
-> 근거리 통신망 (한정된 좁은 지역에 구성된 네트워크)
집, 사무실, 학원 등등 하나의 LAN이라 할 수 있음
초기 투자 비용이 높지만 유지 보수비 낮음

* WAN (Wide Area Network)
-> 원거리 통신망 (넓은 지역을 연결하는 네트워크)
  LAN과 LAN을 서로 연결하는 광역 네트워크
  초기 투자 비용이 낮지만 유지 보수비가 LAN보다 높음

  **통신방식에 따른 네트워크 분류**
  - 유니캐스트 <br>
일대일 전달 방식
수신 측이 한 곳으로 정해져 있는 경우
정확한 특정 목적지의 주소 하나만 가지고 일대일로 전달하는 방식
  - 브로드캐스트 <br>
일대전체 전달 방식
불특정 다수에게 전부 전송하는 경우
  - 멀티캐스트 <br>
일대그룹 전달 방식
정해진 특정 그룹으로 전송하는 경우
특정 다수에게 전송하는 방식

**네트워크의 주소 체계** <br>
각 장비들을 정확한 통신을 위해 네트워크 상에서 서로 구분해야 한다
<br>
-  MAC address <br>
TCP/IP 프로토콜을 사용하는 네트워크에서는 IP address를 사용하여 통신
네트워크 장비의 인터페이스는 고유의 MAC address를 가지고 있다 (NIC 카드, 라우터, 스위치 등)
<br>

- IP address <br>
PC의 주소 -> IP라 부름
이진수 32bit로 구성된 주소 체계
8bit씩 4octet으로 구분, 각 octet을 10진수로 변환해서 표현한다
논리적 주소라고 부른다
  br>

- ARP (Address Resolution Protocol)
IP주소와 MAC주소를 서로 매핑하는 절차를 ARP라고 한다 <br>
Broadcast로 동작 <br>
-> ARP Request <br>
PC에서 게이트웨이 라우터에게 어떤 IP의 맥 어드레스 물어봄 -> 해당 IP가 받고 MAC 주소를 답변해줌, 다른 PC들은 자신의 IP가 아니면 폐기
