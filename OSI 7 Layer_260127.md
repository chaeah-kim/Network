# OSI 7 Layer
<br>
- 통신이 일어나는 과정을 7단계로 구분해서 정리한 것 <br>
- 데이터의 흐름이 한눈에 보인다 <br>
- Trouble shooting이 쉽다 <br>
- 계층을 7단계로 구분하고 각 층별로 표준화를 했기 때문에 여러 회사 장비를 사용해도 네트워크가 이상 없이 돌아간다 <br>

### * OSI 7 Layer 표

- 7 : 응용 계층 <br>
  →  직접 데이터에 엑세스하고 정보 생산, 처리 <br>
  →  사용자와 인터페이스 (ex.인터넷 익스플로러) <br>
  →  사용자와 가장 가까운 프로토콜 정의 <br>
  →   HTTP(80), FTP(21), Telnet(23), SMTP(25), DNS(53), TFTP(69), SNMP(161), DHCP(67)

- 6 : 표현 계층 <br>
→ 정보를 어떤 형식으로 표현 할 것인지 결정 <br>
→ 응용 계층에서 생산한 데이터의 표현, 압축, 보안 기능 제공 <br>
→ 데이터 변환, 복구 <br>
→ 암호화, 복호화 <br>

- 5 : 세션 계층 <br>
→ 응용 프로그램간 연결을 성립하고, 연결이 안정되게 유지되도록 함, 작업 완료 후 연결을 끊는 역할 <br>
→ 통신을 하는 두 호스트들 사이에 세션을 열고 닫고 유지, 종료 관리하는 기능을 담당 <br>

- 4 : 전송 계층 <br>
→ 송신 측에서 전송할 데이터를 안전하고 정확하게 수신측으로 전송하는 기능 <br>
→ 송신 측에서 데이터를 전송 가능한 크기로 세분화, 순서 번호를 삽입 <br>
→ 수신 측에서는 데이터를 재조립, 응용프로그램에 보낸다 <br>
→ 전송한 데이터가 수신 측에 정확히 전송되었는지 오류 제어를 하고 오류 검출 시 재전송을 통해 오류 정정 <br> 
→ Layer4 프로토콜: TCP, UDP <br>

TCP: 신뢰성, 연결지향성, 느림 <br>
UDP: 비신뢰성, 비연결지향성, 빠름 <br>

- 3 : 네트워크 계층 <br>
→ 네트워크상의 주소 체계를 설정하여 네트워크상에서 구별될 수 있도록 한다 <br>
→ 네트워크계층 라우터 : 서로 다른 네트워크와 연결 <br>
(모든 경로에 대한 정보를 라우팅 테이블에 저장) <br>
(라우터끼리 신호를 주고 받음  (라우터끼리 주고 받는 프로토콜 있음) <br>
→ 이러한 주소 정보를 가지고 목적지까지의 경로를 설정, 라우팅 프로토콜을 이용해서 최적 경로 선택 <br>
→ 데이터가 목적지까지 전달되기 위해 어떤 물리적 경로를 거쳐야 하는지 결정하는 역할 <br>
→ IP는 네트워크 계층의 대표적인 프로토콜이다 <br>
<details> 
  <summary> ARP, RARP, ICMP, IGMP </summary>
  - ARP → 논리적 주소를 물리적 주소로 변환시켜주는 기능 <br>
  - RARP → ARP의 반대개념, 물리적 주소(MAC)을 논리적 주소(IP)로 변환 <br>
  - ICMP → Internet Control Message Protocol <br>
   메시지 프로토콜, 정보를 수집하는 용도로 사용 가능, 자신에게 icmp가 들어오는 것은 기본적으로 설정되어 있지 않다
  IP프로토콜은 신뢰성 없는 비연결형 프로토콜이고 에러 발생 원인이나 진단 기능 및 상황 정보를 지원하지 않는데 ICMP는 이것을 보완시켜줌 <br>
  - IGMP → Internet Group Management Protocol <br>
    그룹을 관리하기 위한 프로토콜
</details>

- 2 : 데이터링크 계층 <br>
→ Bit를 Byte로, Byte를 Frame으로 결합 <br>
→ 스위치, 브릿지 (스위치가 라우터 기능 포함하는 것은 L3스위치) <br>
스위치 → 목적지 주소를 확인해 해당 포트로만 전송, 네트워크의 트래픽을 줄일 수 있다 <br>
→ 물리적 회선에 흘러다니는 비트들의 흐름을 제어 <br>
→ 직접 연결되어 있지 않은 네트워크에 대해서는 상위 계층에서 오류 제어를 담당해야 한다 <br>
→ 비트 단위의 전송 오류를 검출하여 정정하는 기능 수행 <br>

- 1 : 물리 계층 <br>
→ 네트워크 통신을 위한 물리적인 표준 정의 <br>
→ 물리적인 인터페이스나 전압의 레벨 등을 결정  <br>

### 계층별 네트워크 장비 <br>

- 물리 계층 (Layer1) <br>
  리피터 (Repeater): 감쇠된 디지털 전기 신호를 증폭하고 재생하여 전송 거리를 연장 <br>
  허브 (Hub): 여러 대의 PC를 연결하여 네트워크를 집중 시킴. 들어온 신호를 연결된 모든 포트로 뿌리기 때문에 충돌 발생 쉬움 <br>

- 데이터 링크 계층 (Layer2) <br>
  브리지 (Bridge): 두 개의 LAN을 연결하여 확장하며 포트가 적은 편 <br>
  스위치 (Switch): 브리지의 발전된 형태. 각 포트에 연결된 장치의 MAC 주소를 테이블에 기록하여 목적지 주소를 가진 포트로만 프레임을 전달. 불필요한 트래픽 줄이고 효율 높임 <br>

- 네트워크 계층 (Layer3) <br>
  라우터 (Router): 서로 다른 네트워크를 연결합니다. 라우팅 테이블을 참조하여 패킷이 목적지까지 가는 가장 최적의 경로 결정합니다 <br>

- 응용 계층 (Layer7) <br>
  게이트웨이 (Gateway): 서로 다른 프로토콜이나 운영체제를 사용하는 네트워크 간을 연결하는 통로 역할을 한다. 모든 계층을 아우르는 복합적인 장비 <br>

<br>

### 계층별 PDU (Protocol Data Unit) <br>
  데이터가 각 계층을 지나면서 불리는 이름 <br>
  1계층: Bit (비트) <br>
  2계층: Frame (프레임 <br> 
  3계층: Packet (패킷) <br> 
  4계층: Segment (세그먼트) <br>
