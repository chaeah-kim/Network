# OSI 7 Layer
<br>
- 통신이 일어나는 과정을 7단계로 구분해서 정리한 것 <br>
- 데이터의 흐름이 한눈에 보인다 <br>
- Trouble shooting이 쉽다 <br>
- 계층을 7단계로 구분하고 각 층별로 표준화를 했기 때문에 여러 회사 장비를 사용해도 네트워크가 이상 없이 돌아간다 <br>

### * OSI 7 Layer 표

- 7 : 응용 계층 <br>
  →  직접 데이터에 엑세스하고 정보 생산, 처리 <br>
  →  사용자와 인터페이스 (ex.인터넷 익스플로러) <br>
  →  사용자와 가장 가까운 프로토콜 정의 <br>
  →   HTTP(80), FTP(21), Telnet(23), SMTP(25), DNS(53), TFTP(69), SNMP(161), DHCP(67)

- 6 : 표현 계층 <br>
→ 정보를 어떤 형식으로 표현 할 것인지 결정 <br>
→ 응용 계층에서 생산한 데이터의 표현, 압축, 보안 기능 제공 <br>
→ 데이터 변환, 복구 <br>
→ 암호화, 복호화 <br>

- 5 : 세션 계층 <br>
→ 응용 프로그램간 연결을 성립하고, 연결이 안정되게 유지되도록 함, 작업 완료 후 연결을 끊는 역할 <br>
→ 통신을 하는 두 호스트들 사이에 세션을 열고 닫고 유지, 종료 관리하는 기능을 담당 <br>

- 4 : 전송 계층 <br>
→ 송신 측에서 전송할 데이터를 안전하고 정확하게 수신측으로 전송하는 기능 <br>
→ 송신 측에서 데이터를 전송 가능한 크기로 세분화, 순서 번호를 삽입 <br>
→ 수신 측에서는 데이터를 재조립, 응용프로그램에 보낸다 <br>
→ 전송한 데이터가 수신 측에 정확히 전송되었는지 오류 제어를 하고 오류 검출 시 재전송을 통해 오류 정정 <br> 
→ Layer4 프로토콜: TCP, UDP <br>

TCP: 신뢰성, 연결지향성, 느림 <br>
UDP: 비신뢰성, 비연결지향성, 빠름 <br>

- 3 : 네트워크 계층 <br>
→ 네트워크상의 주소 체계를 설정하여 네트워크상에서 구별될 수 있도록 한다 <br>
→ 네트워크계층 라우터 : 서로 다른 네트워크와 연결 <br>
(모든 경로에 대한 정보를 라우팅 테이블에 저장) <br>
(라우터끼리 신호를 주고 받음  (라우터끼리 주고 받는 프로토콜 있음) <br>
→ 이러한 주소 정보를 가지고 목적지까지의 경로를 설정, 라우팅 프로토콜을 이용해서 최적 경로 선택 <br>
→ 데이터가 목적지까지 전달되기 위해 어떤 물리적 경로를 거쳐야 하는지 결정하는 역할 <br>
→ IP는 네트워크 계층의 대표적인 프로토콜이다 <br>
